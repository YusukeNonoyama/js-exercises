## 問題

- npm install すると作成される package-lock.json はどのような役割を持つのか。また、リポジトリにコミットすべきか、について説明しなさい。

## 解答

- package-lock.json は package.json で範囲指定されたバージョンの中からインストールされたバージョンを記録するもの
- `npm ci` を実行すると package-lock.json で指定された厳密なバージョンでパッケージがインストールされるため、チームメンバー間での同一の依存関係バージョンの保証ができる
- 開発環境では柔軟な柔軟な依存関係管理のため`npm i`が向いており、本番環境や CI/CDのように再現性が求められる場面での仕様には`npm ci`が適している
- いずれにしても実際にインストールしたパッケージのバージョンをシェアする必要はあるためpackage-lock.json はコミットした方が良い
