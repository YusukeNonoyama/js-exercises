## 15.4-10.11 では #/ や #/active といった URL を利用した。少し昔だとこのような URL は #!/ や #!/active と ! を付けることもあった (もしかしたら職場でそのようなコードを見るかもしれない)。このような形式を当時は hashbang と呼んだ。どうしてこのようなスタイルが存在したのだろうか。参考: [Twitter がページ表示時間を 5 分の 1 に高速化。どのようなテクニックを使ったのか？](https://www.publickey1.jp/blog/12/twitter51.html)

* ページ遷移せずに非同期でサーバーからの情報を更新するページを、google のクローラーにインデックスされるためのものだった
  * 関連用語
    * Ajax; Asyncronous JAvaScript Xml: ページ遷移せずに非同期でサーバーからの情報を更新する技術
    * SPA; Single Page Application
  * hashbang(`"!`) はクロール可能な Ajax アプリとして認識させるための文字列
* pushState() による履歴記録の仕組みによって置き換えられた
  * URL に不自然な"#"を入れなくてよい
  * 通常の URL 形式のため SEO 対応が容易
  * 履歴の管理やURLの書き換えに関してより多くの制御が可能
